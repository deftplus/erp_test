Справочник "Сценарии"

Сценарии используются как средство логического объединения отчетов в комплекты.

Сценарий характеризуется наименованием. Может быть указана также периодичность сценария, однако возможно в рамках одного сценария формировать экземпляры отчетов за периоды разной периодичности.

Для сценария, в рамках которого фиксируются плановые данные (бюджеты), может быть указан сценарий фактических данных: значения показателей отчетов, сформированных в рамках этого сценария, используются для актуализации бюджетов. 

Автоматическое изменение симметричных показателей ВГО.
Для экземпляров отчетов, относящихся к этому сценарию, при их сохранении будет производиться попытка записать симметричные показатели ВГО соответствующих экземпляров отчетов.

Табличное поле "Свойства"

Табличное поле содержит колонки "Свойство" и "Значение свойства".

Обеспечивается добавление и подбор свойств и их значений.

Для добавления нового свойства, изменения состава и типов свойств  следует нажать кнопку "Настройка свойств" командной панели формы списка.

Определение значений свойства зависит от его уникальности.  Если свойство является уникальным, то может быть определен только один экземпляр этого свойства (например, юридический адрес у организации). Для неуникального свойства может быть добавлено произвольное количество значений (например, контактные телефоны). Добавление нового значения неуникального свойства производится в форме списка. При добавлении нового значения свойства обеспечивается автоматическая подстановка типа, определенного при настройке.

Для значений свойств, имеющих тип "Справочник Субконто", контролируется уникальность по значению свойства.

Отображение свойств

Основное свойство в карточке элемента отображается всегда (независимо от того указано для него значение или нет). Свойство, не являющееся основным, отображается в случае, если для него указано значение.

Основные свойства располагаются над дополнительными свойствами.

Строки табличного поля упорядочены по алфавиту наименований свойств.
