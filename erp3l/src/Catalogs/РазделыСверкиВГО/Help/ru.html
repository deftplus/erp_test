Справочник "Разделы сверки ВГО"

Справочник предназначен для создания настройки в Шаблоне сверки ВГО, на основании которой определяются правила сверки разных типов внутригрупповых операций (ВГО).

Если по данному типу ВГО нет необходимости выоплнять сверку, то ставится соответствующий флажок.

В справочнике определяется способ элиминации, применяемый к данному типу ВГО:

 

  • Только сверка – выполняется только сверка, элиминация осуществляться не будет
  • Элиминация статей баланса – сверяются суммы сальдо конечного, которые затем элиминируются
  • Элиминация услуг – сверяются суммы доходов и расходов (обороты) оказанных и полученных услуг, которые затем элиминируются
  • Элиминация запасов – проводится элиминация по основному значению сверки, и корректируется нереализованная прибыль
  • Элиминация показателей отчетов – проводится элиминация показателей отчетов при сверке и элиминации без счетов учета, на основании показателей экземпляров отчетов

     

    На закладке "Аналитики" задается набор аналитик сверки и элиминации, который во многом зависит от способа элиминации и изменяется при его смене. Колонка "Сверка" показывает, какие аналитики используются для сверки. Колонка "Тип для элиминации" определяет роль аналитики в алгоритмах сверки и элиминации.

    На закладке "Ресурсы" указываются значения сверки и элиминации. Колонка "Тип для элиминации" показывает, как использовать значение ресурса: в качестве показателя сверки или значения себестоимости.

    На закладке "Источники данных" описываются источники, из которых берутся значения сверки, аналитики и значения элиминации.

    Колонка "Способ определения направления учета" необходима для того, чтобы разделить данные дебитора и кредитора. Можно выбрать из перечисления фиксированное для источника данных значение: Только дебиторы, Только кредиторы. Или определять По знаку операции: больше нуля – дебитор, меньше нуля – кредитор.
    В поле "Источник данных" указывается вид отчета, позволяющий указать, как получать данные для сверки и элиминации.

    На закладке "Автоматическое урегулирование" в поле "Порог существенности, %" можно задать процент, ниже которого отклонения в сверке будут урегулироваться автоматически.
