Справочник "Этапы процессов"

Назначение

Справочник "Этапы процессов" содержит список этапов, входящих в состав рабочих процессов подготовки отчетности. Справочник "Этапы процессов" подчинен справочнику "Шаблоны процессов".

Содержимое этапа представляет собой последовательность видов отчетов или ручных корректировок, которые должны быть сформированы на текущем этапе. В рамках одного этапа могут формироваться только документы одного вида: либо экземпляры отчетов, либо ручные корректировки.

Помимо вида документов, для этапа определяются:

  • плановая длительность этапа (в днях);
  • тип организации (обычная, элиминирующая, консолидирующая), к которой применим данный этап;
  • основная валюта документов этапа (экземпляров отчета или ручных корректировок).

На закладке "Формируемые отчеты" ("Ручные корректировки") приводится перечень видов отчетов или шаблонов ручных корректировок, формируемых на текущем этапе. Учитывая, что отчеты и ручные корректировки, формируемые на этапе, могут учитывать данные других документов, сформированных на том же этапе, перечень видов отчетов или шаблонов ручных корректировок должен быть приведен именно в том порядке, в котором должны быть сформированы документы.

Для каждого из видов отчетов на закладке "Формируемые отчеты" определяется способ формирования отчета:

  • ручное заполнение;
  • импорт извне;
  • автоматическое заполнение по процедуре заполнения;
  • автоматическая консолидация;
  • автоматическое свертывание по аналитикам на уровне отчета.

Установленный флаг "Автоматическое утверждение" указывает на автоматического утверждение отчета текущей строки в ходе выполнения процесса. Если флаг установлен и проверяющий текущего отчета соответствует ответственному за этап, либо один из этих реквизитов не указан, то при попытке утвердить этап в ходе выполнения процесса отчеты утверждаются автоматически. Флаг удобно использовать, если количество сотрудников, вовлеченных в процесс обработки отчетности невелико.

В зависимости от способа формирования отчетов, дополнительно могут быть определены:

  • для отчетов, формируемых в результате импорта - флаг "Отображать сравнение", позволяющий указывать поведение прикладного решения в случае возникновения ошибок при импорте. При пакетном импорте отчетов, у которых установлен флаг "Отображать сравнение", в случае возникновения ошибок, связанных с контролем правильности вычисления ячеек отображается форма сравнения макетов, позволяющая визуально идентифицировать ошибочный показатель. Далее пользователю предоставляется возможность принять перевычисленные системой значения показателей отчетов, либо отказаться от сохранения отчета, оставив его для детального анализа. При этом отчет остается в состоянии "Выполняется";
  • для отчетов, формируемых в результате сворачивания по периметру - периодичность сворачиваемых отчетов.

На закладке "Виды операций" для этапа, на котором формируются операции, могут быть указаны шаблоны обычных операций, операций обратного распределения и элиминации. Типы используемых шаблонов зависят от типа организации этапа. Как и для видов отчетов, для шаблонов операций определяется значение флага "Автоматическое утверждение".

Для операций элиминации, выплняемых на этапе, дополнительно определяются тип организаций для сверки (обычные или консолидированные) и сценарий, на который будут отнесены расхождения, не урегулированные автоматически.

На закладке "Этапы предшественники" могут быть указаны один или несколько этапов, после завершения которых начинается выполнение текущего этапа.

На закладке "Организационные единицы" определяется область действия этапа (на какие организации он распространяется). Область действия определяется перечнем профилей организаций или перечнем периметров. Если в область действия этапа включен профиль организации, это значит, что экземпляры отчета (ручные корректировки) будут формироваться для всех организаций профиля. Если в область действия этапа включен периметр, перечень организаций, для которых формироваться экземпляры отчета или ручные корректировки, зависит от типа организаций этапа:

  • тип организаций этапа "обычная": для обычных и консолидирующих организаций, входящих в состав периметра;
  • тип организаций этапа "элиминирующая": для элиминирующих организаций, входящих в состав периметра;
  • тип организаций этапа "консолидирующая": для консолидирующей организации, с которой ассоциируется периметр (собственно, для той организации, которая указана на закладке в качестве периметра).

В колонке "Ответственный за этап" для каждого профиля (периметра) указывается сотрудник управляющей компании (центра консолидации), ответственный за формирование отчетов этапа.

На закладке "Дополнительные валюты" указываются дополнительные валюты, которые при выполнении процесса назначаются всем отчетам данного этапа.

Для этапа, предназначенного для формирования экземпляров отчетов, может быть настроена рассылка комплекта отчетности этапа. На одноименной закладке определяется список контактных лиц - сотрудников организаций группы, которым автоматически рассылается комплект отчетов этапа. Всем контактным лицам, включенным в рассылку, направляется комплект отчетов в момент утверждения этапа.

Отдельным контактным лицам может высылаться также предварительная отчетность, формируемая в ходе выполнения регламентного задания, которое настраивается из формы этапа процесса при нажатии кнопки "Настройка выполнения текущего этапа по расписанию". В ходе выполнения регламентного задания автоматически формируются и заполняются экземпляры отчетов (ручные корректировки этапов). При этом отчеты и операции остаются в состоянии "Выполняется", их автоматическое утверждение не производится.
