Лот

Используется только в составе закупочной процедуры. Содержит требования к закупке однотипной продукции (товаров, работ, услуг) с целью проведения торгов и приобретение продукции по оптимальной цене в соответствии с четко установленными критериями. Лоты вводятся на основании строк плана закупок. Для организаций, ведущих учет по 223-ФЗ, на основе одной сроки плана закупок может быть создан только один лот. То есть строка плана закупок попадает в лот целиком и не может быть разделена на два лота. Каждый лот содержит исходную информацию из строки плана закупок и уточняется в ходе подготовки к закупочной процедуре. В результате закупочной процедуры по каждому лоту заключают договора с победителями. 

Графическая схема справки 

С особенностями оформления и использования объектов подсистемы Управление закупками можно ознакомиться на схемах:
- Корпоративные закупки
- Аккредитация поставщиков 
- Подготовка и проведение торгов 
С другими схемами системы можно ознакомиться по ссылке.

Особенности заполнения реквизитов

В основном варианте использования лот создается из закупочной процедуры по строке плана закупок. Закупочная процедура содержит достаточно много одинаковых реквизитов с лотом. Эта особенность связана с многолотовой закупкой. При работе с лотом действует следующее правило. Закупочная процедура содержит общие для всех лотов условия, а лот частные. То есть реквизиты лота имеют более высокий приоритет перед аналогичными реквизитами закупочной процедуры. Поэтому при создании многолотовой закупочной процедуры настоятельно рекомендуется сначала максимально полно заполнить общие поля закупочной процедуры, а потом добавлять лоты. В этом случае лот примет значения общих полей закупочной процедуры и пользователю останется только отразить особенности конкретного лота. Общие для всех лотов поля закупочной процедуры выделены зеленым фоном.

Для однолотовой закупочной процедуры порядок заполнения реквизитов не имеет принципиального значения. Если пользователь проигнорировал порядок заполнения полей, то общие поля можно заполнить из лота по кнопке "Заполнить" - "Заполнить из закупочной процедуры". При заполнении полей лота обратите внимание на метод оценки. По умолчанию метод оценки выставляется от способа закупки и только в лоте его можно изменить. При выборе метода оценки нужно помнить, что метод оценки должен быть одинаковый во всех лотах закупочной процедуры. При записи система проверит метод оценки для многолотовой закупочной процедуры.  Метод оценки по умолчанию принимает следующие значения:

По критериям:

  • Закупка у единственного поставщика;
  • Конкурс;
  • Запрос предложений;
  • Конкурентные переговоры;
  • Конкурс в электронной форме у МСП;
  • Запрос предложений в электронной форме у МСП;

По ценам:

  • Запрос котировок;
  • Запрос котировок в электронной форме у МСП;
  • Аукцион;
  • Аукцион в электронной форме у МСП;

Наименование лота по умолчанию выставляется в порядке создания (добавления) и имеет представление "Лот №1", "Лот №2" итд. Рекомендуем при создании поменять на осмысленное наименование, связанное с предметом закупки или принятой в организации кодировкой лотов. Если наименование лота оставить без изменения, то в системе будет проблематично отличить один лот от другого.

Табличная часть лота недоступна на редактирование. Исключение сделано только для случая экстренной закупки при условии, что Организация-Заказчик не ведет учет по 223-ФЗ. При наличие признака экстренной закупки (флажок "Экстренная закупка" в закупочной процедуре) табличная часть заполняется данными справочника номенклатуры. Система контролирует не нулевое значение количества и суммы лота. Если в настройках системы включен флажок "Разделять номенклатуру по товарным категориям", то в подборе будет доступна номенклатура из указанной в закупочной процедуре товарной категории. 

Вид закупки и его влияние на процесс выбора поставщика

В лоте может быть выбрано одно из трех значений реквизита вид закупки. Вид закупки определяет, разрешено ли участникам подавать неполные предложения, а также возможность Организации - Заказчика по выбору победителя. В одном случае победитель может определяться по цене позиции, в другом - по цене лота.

Лотовая закупка

Лот торгуется целиком. То есть от поставщиков принимают только предложения, покрывающие все позиции лота. Данный вид закупки может иметь только одного победителя, предложившего наилучшие условия поставки всех позиций лота. 

Попозиционная закупка

Лот торгуется по отдельным позициям. То есть от поставщиков принимают как полные предложения, так и предложения на поставку только выборочных позиций. Побеждает победитель, предложивший наилучшие условия по позиции. Данный вид закупки может иметь несколько победителей.

Зонтичная закупка

Лот также торгуется по позициям, но заказчик на подготовительном этапе определяет количество победителей и условия распределения между ними объема поставки. Объем поставки зависит от места участника. Чем лучше предложение участника, тем больший объем он будет поставлять. Условие распределения задается на закладке "Условия зонтичной закупки", а вид закупки и количество победителей на закладке  "Основные сведения"лота. Для товарной категории, не имеющей количества, распределение задается в процентах, для номенклатуры - в натуральных величинах.
