Справочник "Периоды"

Назначение справочника

Справочник "Периоды" предназначен для хранения нормализованных интервалов времени в целях упрощения их использования.

Дерево справочника

Справочник "Периоды" представляется для редактирования и выбора в виде иерархического списка - дерева.

Иерархия справочника

Справочник "Периоды" - иерархический.

Короткие периоды могут быть подчинены более длинным периодам при условии своего полного попадания внутрь длинных периодов. Это условие проверяется при записи нового элемента справочника в информационную базу.

Автоматическая генерация периодов

Кнопка "Заполнить" командной панели формы списка справочника вызывает на экран вспомогательную форму "Генерация новых периодов".

С помощью этой формы можно автоматически сформировать комплекты периодов разной длительности внутри более длинного периода.

Процедура автоматического заполнения справочника "Периоды" позволяет сформировать дерево подчиненных элементов-периодов выбранного диапазона периодичности, начиная от заданного элемента.

Для заполнения указываются:

  • родительский элемент-период;
  • периодичности элементов в подчиненных уровнях.

Реализована опциональная поддержки уровня генерации вложенных периодов, например: год поквартально и помесячно, или год только помесячно.

При автоматическом заполнении создаваемый период переносится в нужный уровень иерархии периодов, что обеспечивает возможность последующего добавления промежуточных периодов. Например, сначала можно для выбранного года сформировать 12 помесячных периодов, затем добавить 4 квартальных периода.

Редактирование периода

При редактировании нового элемента достаточно указать дату начала периода и периодичность. Дата окончания и наименование периода подставятся автоматически. Для произвольных периодов возможно ручное определение даты конца периода.

Проверка при записи элемента

При записи элемента осуществляются следующие проверки и установки.

  • Проверка на соответствие даты начала, периодичности и даты окончания:
    • дата начала приводится к одному из значений, возможных для данной периодичности;
    • дата окончания рассчитывается в соответствии с датой начала и периодичностью.
  • Проверка вложенности периодов по иерархии:
    • дата начала не может быть меньше, чем дата начала периода-родителя, если он указан;
    • дата начала не может быть больше, чем даты начала дочерних элементов;
    • дата окончания не может быть больше, чем дата окончания периода-родителя, если он указан;
    • дата окончания не может быть меньше, чем даты окончания дочерних элементов.
  • Проверка на уникальность периодов:
    • в справочнике не должно быть элементов с одинаковыми датами начала и периодичностью.
  • Проверка на неизменность периода:
    • если существуют ссылки на период, дата начала и периодичность не должны изменяться.

Свойства справочника

Наименование

В наименовании элемента справочника "Периоды" указывается представление элемента.

Наименование формируется автоматически при записи элемента.

Реквизиты справочника

Дата начала

В реквизите "Дата начала" указывается дата начала периода.

Дата окончания

В реквизите "Дата окончания" указывается дата окончания периода.

Периодичность

В реквизите "Периодичность" указывается длительность периода, которая выбирается из перечисления "Периодичность".

Произвольный

Если реквизит "Произвольный" заполнен, то дата начала и окончания периода могут быть указаны вручную. При этом не контролируется соответствие даты начала, периодичности и даты окончания периода, однако осуществляются проверки вложенности и уникальности произвольного периода. Произвольные периоды используются, например, в случаях, если финансовый период не совпадает с календарным.
