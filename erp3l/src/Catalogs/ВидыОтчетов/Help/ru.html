Справочник "Виды отчетов"

Справочник "Виды отчетов" содержит самые общие сведения о формируемом отчете. Основной объем информации о структуре отчетов, а также о правилах их заполнения, содержится в остальных справочниках, которые подчинены справочнику "Виды отчетов":

  • "Строки отчетов";
  • "Колонки отчетов";
  • "Группы раскрытия показателей";
  • "Показатели отчетов";
  • "Бланки отчетов";
  • "Правила обработки";
  • "Правила проверки".

Для работы с данными различных справочников, относящимися к определенному виду отчета, предназначена форма "Структура вида отчета". Эту форму можно открыть из формы списка справочника "Виды отчетов" в 2 шага:

  • в списке отчетов выделите запись нужного вида отчета;
  • нажмите на кнопку "Структура" в командной панели формы списка.

Свойства и реквизиты справочника "Виды отчетов"

В качестве кода элемента справочника "Виды отчетов" указывается краткое уникальное имя вида отчета. Код используется в выражениях для обращения к этому отчету, поэтому он не должен содержать некоторых специальных символов.

В реквизитах "Вид аналитики 1", "Вид аналитики 2" и т. д. следует указать виды субконто, которые могут быть использованы для указания аналитики на уровне отчета.

Примечания
1. Вид аналитики, указанный на уровне отчета, распространяется на все показатели этого отчета — подчиненные элементы справочника "Показатели отчетов".
2. Если для вида отчета задан вид аналитики, это означает следующее: отчет можно сформировать как для конкретного значения аналитики, так и без детализации по аналитике — так называемый "свернутый по аналитикам" отчет, в котором объединены все возможные значения вида аналитики. Можно формировать комплекты детализированных отчетов: первый отчет комплекта — для первого значения аналитики, второй отчет комплекта — для второго значения аналитики и т.д.
3. На уровне отчета все виды аналитики можно не устанавливать. При этом появляется возможность задавать собственные виды аналитики для показателей отчетов так, чтобы общее количество видов аналитики для отдельного показателя (как заданных на уровне отчета, так и заданных на уровне конкретного показателя) не превысило 3 (или 5 для варианта "1С:Консолидация 8. Проф".
