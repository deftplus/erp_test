Форма предназначена для настройки процесса оценки по этапам и критериям. Используется при оценке инвестиционных проектов и оценке предложений поставщиков подсистемы централизованное управление закупками.

Описание объектов формы

В верхней табличной части Этапы оценки вводятся этапы процесса, ниже располагается зависимая табличная часть Критерии оценки. Одному этапу может быть назначено множество критериев. При позиционировании на строке табличной части Этапы оценки в табличной части Критерии оценки отображаются только критерии соответствующие выделенному этапу. В нижней части формы отображается зависимая табличная часть Эксперты, назначенные выбранному критерию зависимая от табличной части Критерии оценки.  При назначении действует правило: эксперты из состава комиссии назначаются на оценку каждого критерия в соответствии с компетенциями по функциональному направлению. Это означает, что если у критерия поле функциональное направление не заполнено, то его может оценивать любой эксперт из состава комиссии. Назначение на оценку получают все члены комиссии. Если у критерия поле функциональное направление заполнено, то оценку критерия может провести только эксперт данного функционального направления. Состав комиссии определяется на вкладке Экспертная комиссия. Один или несколько экспертов должны закрывать этап оценки. Настройка процесса считается некорректной, если присутствуют критерии, по которым не назначен эксперт для оценки или ни один эксперт не назначен закрывающим этап оценки. 

  • Разрешена оценка объектов после завершения процедуры выбора - позволяет довнести данные после завершения процедуры. Флажок установаливается только при выборе в списке этапов предопределенного этапа Последующая оценка. Позволяет после выбора наилучшего предложения по предопределенному этапу провести дооценку по критериям этапа. Этап отображается в отчете интегральная оценка.   

Настройка критериев оценки

Каждый критерий содержит набор реквизитов, по которым выполняется его оценка.

  • Вес - значимость критерия в составе общего перечня критериев этапа оценки. Совокупный вес всех критериев по этапу должен составлять 1 (100%).
  • Мин. - минимальное значение оценки, назначенное данному критерию , которое может выставить эксперт.
  • Макс. - максимальное значение оценки, назначенное данному критерию , которое может выставить эксперт.
  • Рассчитать значение -  при установленном флажке значение критерия получается расчетным путем по источнику данных.
  • Источник данных -  в качестве источника может выступать:
    • показатель текущей или внешней информационной базы;
    • показатель группы раскрытия текущей или внешней информационной базы;
    • регистр накопления текущей или внешней информационной базы;
    • регистр сведений текущей или внешней информационной базы;
    • произвольный запрос к текущей или внешней информационной базы;
    • проводка трансформационной корректировки;
    • справочник или документ текущей или внешней информационной базы.
  • Не использовать баллы - будут использованы фактические значение оценки, приведение к баллам не выполняется.
  • Балл 0 от [] до[] - задает диапазон значений оценки соответствующий указанному баллу.

Использование шаблонов заполнения

Вызов шаблона выполняется из командного меню (зеленая стрелочка). Шаблон содержит дополнительные разделители (варианты заполнения шаблона), которые позволяют:

  • заполнить процесс целиком;
  • заполнить только этапы и критерии;
  • заполнить только экспертов.

    • Шаблон перед применением может быть просмотрен в форме шаблоны заполнения по кнопке Открыть шаблон. Откроется предварительная форма настройки процесса, заполненная по шаблону. После применения шаблона отменить его действие нельзя.

    Создание шаблона выполняется на основании настроенного процесса из командного меню (красная стрелочка). При сохранении необходимо указать название шаблона и вариант заполнения шаблона. 

    Управление процессом выбора оценки предложений поставщиков 

    Доступ к процессу оценки эксперты получают после перехода лота в состояние Работа закупочной комиссии. На данном этапе в системе зафиксированы предложения поставщиков и можно переходить к выбору наилучшего предложения. Следует учитывать, что запуск процесса и оценка по критериям выполняется только для лотов, у которых в форме лота поле метод оценки предложений поставщиков установлен в значение по критериям. Оценку предложения эксперт может выполнить из списка лотов по кнопке меню Выбор альтернатив -> Ввести значения критериев. Но основной выход на оценку выполняется из раздела Главное -> Оценка альтернатив. В этом варианте эксперту доступны все критерии по лоту, назначенные ему для проведения оценки. Выполнив оценку эксперт закрывает форму. Для отображения информации по исполнению процесса оценки предназначен отчет Интегральная оценка альтернатив. Эксперту доступны только критерии этапа, по которому выполняется оценка. После оценки экспертами этапа, ответственный за завершение оценки этапа переводит процесс на следующий этап или завершает оценку. Завершить оценку можно в разделе Главное ->Выбор альтернатив.  При необходимости можно рестартовать процесс выбора. Следует учитывать, что при выполнении данной команды введенные значения оценки по процессу очищаются. 

    Управление процессом выбора инвестиционной программы

    Настройка процесса выбора выполняется для раздела инвестиционной программы. В качестве объектов оценки выступают инвестиционные проекты, входящие в раздел инвестиционной программы. Настройка процесса выбора выполняется из формы инвестиционной программы, там же можно ввести оценку проектов и выбор альтернатив. Основной выход на оценку выполняется из раздела Главное -> Оценка альтернатив. В этом варианте эксперту доступны все критерии раздела программы, назначенные ему для проведения оценки. Выполнив оценку эксперт закрывает форму. Для отображения информации по исполнению процесса оценки предназначен отчет Интегральная оценка альтернатив. Эксперту доступны только критерии этапа, по которому выполняется оценка. После оценки экспертами этапа, ответственный за завершение оценки этапа переводит процесс на следующий этап или завершает оценку. Завершить оценку можно в разделе Главное ->Выбор альтернатив.  При необходимости можно рестартовать процесс выбора. Следует учитывать, что при выполнении данной команды введенные значения оценки по процессу очищаются.
