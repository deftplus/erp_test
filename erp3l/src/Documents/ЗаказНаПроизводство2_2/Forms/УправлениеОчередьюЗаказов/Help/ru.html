Управление очередью заказов к производству

Рабочее место предназначено для управления списком заказов на производство, переданных к производству, и осуществления действий по исполнению этих заказов: определение очередности исполнения, формирование этапов производства, контроль обеспечения материалами, планирование графика производства.

В рабочее место попадают заказы в статусе К производству. Заказы в других статусах не попадают в это рабочее место ни с какими отборами. Но если в системе имеются заказы в статусе Формируется, то по гиперссылке Формирующиеся заказы (кол-во) можно увидеть их количество и перейти к форме со списком формирующихся заказов.
Заказы, по которым завершена вся работа, рекомендуется переводить в статус Закрыт, чтобы не загромождали список. Перейти к таким заказам можно по гиперссылке Закрытые заказы.

В рабочем месте установлена сортировка заказов по убыванию очередности планирования. Сверху списка заказы с более высоким приоритетом. Упорядочивание происходит сначала по приоритету заказа, потом по приоритету подразделения-диспетчера, затем по месту заказа в очереди. Заказ, который переводится в статус К производству, попадает в конец очереди в рамках своего приоритета. Поднять заказ в очереди в списке заказов одного приоритета можно при помощи кнопок со стрелками или перетащив заказ мышкой в нужно место очереди.
Возможность изменения очереди недоступна, если в форме установлены отборы.
Пользовательская сортировка списка не предусмотрена.

Список заказов

Список заказов содержит 3 колонки индикаторов состояний заказа: состояние выполнения этапов, обеспеченность материалами, состояние выпуска.

Состояние выполнения этапов производства

Колонка содержит индикаторы необходимых действий над этапами заказа. При двойном щелчке по индикаторы производится действие, соответствующее отображаемой проблеме. Возможные индикаторы в этой колонке:

Требуется сформировать этапы
По заказу не сформированы все необходимые этапы производства. Либо по заказу еще не формировались этапы, либо есть необходимость доформировать. Например, дополнена продукция, или в этап добавлены полуфабрикаты, производимые в процессе. Двойным щелчком по такому индикатору выполняется команда формирования этапов по заказу.

Требуется проверить этапы
Среди этапов заказа есть те, которые не удалось провести в статусе Сформирован, они находятся в статусе Формируется. Такие этапы может потребоваться дозаполнить и отметить сформированными. Двойным щелчком по такому индикатору открывается диспетчирование этапов с отбором по выбранному заказу.

Требуется планировать график производства
Некоторым или всем этапам заказа требуется расчет графика. Такой индикатор показывает, что график производства по этому заказу либо вообще не рассчитывался, либо стал неактуальным и требует перепланирования. По этому индикатору открывается форма планирования графика по заказу.

Требуется передать этапы к выполнению
Индикация наличия по заказу этапов, готовых к выполнению, но не переданных к выполнению. Готовыми к выполнению считаются этапы, у которых нет невыполненных предшественников. Двойным щелчком по индикатору всем готовым к выполнению этапам будет поставлен статус К выполнению

Обеспеченность материалами

В колонке может содержаться индикатор , показывающий наличие этапов, для которых все еще не обеспечены все необходимые материалы. Проблемными являются такие материалы, у которых в этапе стоит действие К обеспечению. Двойным щелчком по индикатору открывается форма диспетчирования этапов с отбором по заказу и необходимости обеспечения. Из этой формы можно найти, в каких именно этапах есть проблемы.

Состояние выпуска

Может содержать индикаторы:

Нарушена дата потребности
Индикатор появляется в том случае, если по графику ожидается опоздание производства всей или части продукции относительно желаемой даты выпуска в заказе.

Все произведено
Индикатор появляется у тех заказов, по которым произведена вся продукция. Такие заказы можно перевести в статус Закрыт.

Отборы

В шапке формы присутствуют отборы по желаемой дате выпуска, приоритету, ответственному. Кроме того, на панели справа от списка присутствуют отборы, соответствующие индикаторам состояний заказа. Отборы по индикаторам при одновременном выборе объединяются по “или”: будут отобраны заказы, в которых горит любой из выбранных индикаторов.

Если выбрать все индикаторы, будут отобраны все заказы кроме тех, у которых не горит ни один индикатор. Чтобы получить список всех заказов к производству, необходимо убрать все отборы.

Команды

Сформировать этапы производства
По этой команде система сформируется необходимые этапы для производства продукции и производимых полуфабрикатов, для которых указаны спецификации. Если для части продукции или полуфабрикатов указаны спецификации, а для части - нет, система сформирует этапы только для тех позиций, для которых есть спецификации. Если при очередном формировании этапов не удастся сформировать ни одного по причине отсутствия спецификаций, система сообщит об этом. Для формирования этапов по таким заказам необходимо либо заполнить спецификации, либо сформировать этапы вручную.

Передать этапы к выполнению
По этой команде передаются к выполнению те этапы, у которых нет невыполненных предшественников. Если есть необходимость заранее поставить статус к выполнению всем этапам, включая тех, у которых есть невыполненные предшественники, можно воспользоваться формой диспетчирования этапов.

Пометить на удаление этапы производства (находится во всплывающем меню)
По этой команде помечаются на удаление все этапы производства, относящиеся к заказу.

Планировать текущий
По этой команде открывается форма планирования графика текущего заказа. При планировании текущего заказа желательно наличие актуального графика производства по всем вышестоящим в очереди заказам. В противном случае последующее перепланирование вышестоящих заказов может сделать неактуальным график этого заказа.

Планировать выбранные
Команда позволяет запланировать график нескольких выбранных заказов. Для выполнения этой команды тоже желательно наличие актуального графика вышестоящих заказов. По этой команде открывается окно настроек параметров планирования, в котором можно выбрать опции: Задействовать резерв доступности, Полное перепланирование, Отменить ручные изменения графика. Назначение этих настроек описано в справке по форме планирования графика.

Планировать очередь по текущий
По этой команде осуществляется планирование всей очереди заказов с самого первого по выбранный. При выполнении этой команды открывается окно настроек параметров, как и в предыдущей команде.

Перенести желаемые даты обеспечения
Команда предназначена для подгонки желаемых дат обеспечения материалами под график производства этапов. Это позволяет сократить случаи преждевременного обеспечения материалами тех этапов, выполнение которых начнется еще не скоро.
При выполнении команды отображается отчет, который позволяет визуально проконтролировать список материалов, с желаемой датой обеспечения раньше даты начала этапа. Из отчета можно открыть интересующие этапы для ручного изменения или же воспользоваться командой “”, которая по всем этапам установит желаемые даты обеспечения равными запланированным по графику датам начала этапов.
При использовании этой команды необходимо понимать, что этапы, запланированные далеко в будущем из-за других ограничений (виды РЦ, предыдущие этапы), для которых подвинуто обеспечение материалами на более поздние даты, впоследствии может не получиться перепланировать на более ранние сроки уже из-за ограничений по материалам.

Закрыть заказ
Выбранные заказы по этой команде переводятся в статус Закрыт. Для удобства перед выполнением этой команды можно установить отбор по индикатору Все произведено. При включенной опции Не закрывать частично отработанные заказы на производство в разделе НСИ и администрирование - Производство, система не даст закрыть заказ, по которому еще не все произведено.

Диспетчирование этапов
Команда отрывает форму диспетчирования этапов производства с установленным отбором по выбранному заказу.
Структура заказа
Открывает форму структуры заказов для выбранных заказов на производство. Позволяет в одном окне увидеть структуру нескольких заказов.

Формирование заказов
Гиперссылка используется для формирования заказов на производство полуфабрикатов, обеспечиваемых отдельно. По этой команде открывается рабочее место Формирование заказов по потребностям с предустановленным типом обеспечения Собственное производство.

Обеспечение цеховых кладовых материалами
Гиперссылка используется для формирования заказов обеспечивающих потребности цеховых кладовых. По этой команде открывается рабочее место Формирование заказов по потребностям с предустановленным отбором по типу склада Кладовая цеха.
