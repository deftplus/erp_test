Строка плана закупок (СПЗ)

Документ предназначен для регистрации основной информации о предмете закупки. На основании СПЗ в дальнейшем формируются лоты. 

Может использоваться как организациями, ведущими закупки по 223-ФЗ, так прочими организациями. Внешний вид формы документа, его реквизитный состав, контроли и поведение документа при проведении зависят от выбранной организации, а именно от реквизита реквизита "Закупка по 223-ФЗ" справочника "Организационные единицы" (закладка "Регламент"). 

Графическая схема справки
С особенностями оформления и использования объектов подсистемы Корпоративные закупки можно ознакомиться на схемах:
- Общая схема: Корпоративные закупки
- Аккредитация поставщиков 
- Подготовка и проведение закупочных процедур 
С другими схемами системы можно ознакомиться по ссылке.

Работа с документом

Строка плана закупок (СПЗ) может создаваться вручную или автоматически обработкой "Помощник формирования программы закупок". Важное значение для последующего процесса представляет собой статус документа. Только утвержденные СПЗ войдут в программу закупок для последующей закупки. 

Для организаций, ведущих учет по 223-ФЗ программа закупок и составляющие её строки плана выгружаются на сайт Госзакупок (ЕИС). Исключение составляют СПЗ с флажком "Не размещается на ЕИС" (малые закупки, гостайна, причины ч15с4 223-ФЗ). Флажок устанавливается автоматически по данным документа. Например, если сумма строки плана не превышает 500 тыс. руб. (константа ) или Окончательное решение принимает пользователь. Флажок доступен для редактирования. 

Ручной режим создания СПЗ

В ручном режиме СПЗ вводится из программы закупок или из формы списка. Источником данных для строки плана может выступать потребность, зарегистрированная документом "Операционный план" для бюджета закупок с видом операции "Планирование". В этом случае табличная часть заполняется подбором из остатков плановой потребности за указанный период и недоступна для редактирования. Цена, Сумма, НДС заполняются по данным Операционного плана. При заполнении СПЗ из потребности пользователь работает с номенклатурой, прошедшей контроль по лимитам бюджета закупок и не имеет возможности добавить позиции из справочника Номенклатура. До открытия подбора из плана потребности необходимо записать СПЗ так как выполняется запрос к остаткам плана потребности по дате и времени документа. 

Если в организации за закупку части номенклатуры отвечают категорийные менеджеры, то есть ведется деление номенклатуры по товарным категориям (включена константа "Разделять номенклатуру по товарным категориям"), то в СПЗ появляется возможность ограничить выбор потребности товарной категорией. При этом указание товарной категории не обязательно. Если товарная категория не указана, то в подборах доступна все номенклатура. В противном случае, подбор ограничен номенклатурой указанной товарной категории.

При необходимости зарегистрировакупкать СПЗ вне плана потребности, на закладке "Основные сведения" необходимо установить флажок "Вне плановой потребности". При установленном флажке табличная часть доступна для ввода данных из справочника "Номенклатура". 

Важно! Строка плана закупок, созданная вне плана потребности, проходит вне лимитов. 

При заполнении табличной части каждая номенклатура или товарная категория сверяется с правилами проведения закупок "Положение о закупках" (Справочник, содержащий правила проведения закупок в организации) и устанавливает прописанные в правилах условия. Например, устанавливает для номенклатуры флажок "Требуется обоснование", а при записи документа выполняется проверка корректно установленного способа закупки, формы проведения и других условий, прописанных в положении о закупках. При наличие флажка "Требуется обоснование" даже в одной позиции табличной части инициирует отображение на форме закладки "Обоснование", в которой пользователь обосновывает расчет НМЦ и требования к проведению закупки. В следствие обоснования СПЗ может потребоваться создание документа "Обоснование требований к закупочной процедуре", который можно создать на закладке "Обоснование" СПЗ. Данный документ не является обязательным для создания и заполнения. Он содержит дополнительную информацию для закупщика и может использоваться на этапе подготовки закупочной процедуры. Обоснование считается завершенным при установленном флажке "Обоснование завершено".  

Автоматический режим создания СПЗ

При работе от плана потребности можно автоматически создать СПЗ в соответствии с правилами, прописанными в справочнике "Положения о закупках" обработкой "Формирование строк плана закупок".  Обработка имеет несколько режимов работы, результатом каждого режима являются созданные или актуализированные СПЗ. Документы, созданные обработкой значительно сокращают трудозатраты при первичном создании плана и особенно при его последующей актуализации. Документы создаются в статусе "Черновик", требуют проверки и корректировки заполненной текстовой информации о предмете договора и общих требованиях к проведению закупки. Обработка умеет не только создавать, но и корректировать существующие СПЗ автоматически, в следствие изменения плана потребности. Корректироваться могут СПЗ, для которых выполняется несколько условий: 

  • Дата официального объявления о закупке прописанная в СПЗ больше даты корректировки (текущей даты);
  • По СПЗ не создан лот в рамках закупочной процедуры. То есть подготовка к закупке еще не началась;
  • В СПЗ включен флажок "Запретить автоматическое изменение документа". Флажок доступен пользователю на любом этапе жизненного цикла документа.

Если в организации за закупку части номенклатуры отвечают категорийные менеджеры, то есть ведется деление номенклатуры по товарным категориям (включена константа "Разделять номенклатуру по товарным категориям"), то в результате работы обработки будут сформированы СПЗ в заданной пользователями группировке, но с дополнительным разделением товарным категориям. То есть каждая СПЗ будет содержать номенклатуру из одной товарной категории.

Особенности отображения реквизитов на форме и поведение при записи для организаций ведущих учет по 223-ФЗ

Если организация ведет учет по 223-ФЗ, что определяется значением флажка в форме организации Заказчика "Закупка по 223-ФЗ", то на форме отображаются реквизиты:

  • Закупка инновационной продукции, высокотехнологичной продукции и лекарственных средств - задает принадлежность СПЗ к инновационной программе закупок.
  • Привлечение субъектов малого и среднего предпринимательства (МСП) в качестве субподрядчиков - указывает, что закупка выполняется среди крупных поставщиков, разрешено для исполнения обязательств привлекать на субподряд организации, относящиеся к МСП.
  • Закупка только у субъектов малого и среднего предпринимательства - указывает на то, что в закупке участвуют только МСП. По такой закупке СПЗ должна содержать план закупки и финансирования на три года. Появляется закладка "Финансирование", а в табличной части "Потребности" периодичность по годам. Длительность планирования определяет реквизит "Дата исполнения договора". При закупке у МСП в момент записи документа выполняется проверка, что план задан на три года, заполнена табличная часть "Финансирование" и в качестве способа закупки выбран один из четырех разрешенных электронных способов закупки.
  • Способы закупки содержат следующий набор значений: 
    • Закупка у единственного поставщика; 
    • Запрос котировок; 
    • Конкурс;
    • Запрос предложений;
    • Конкурентные переговоры;
    • Аукцион; 
    • Конкурс в электронной форме у МСП; 
    • Аукцион в электронной форме у МСП; 
    • Запрос котировок в электронной форме у МСП; 
    • Запрос предложений в электронной форме у МСП; 
  • Закладка ЕИС - содержит дополнительную информацию (регистрационный номер в ЕИС, обоснование внесения изменений при корректировке,  особенности закупки у МСП, порядок формирования цен).

Корректировка СПЗ

Внесение изменений в СПЗ возможно только вводом на основании изменяемой строки плана. При этом в программу закупок попадут только последние утвержденные документы. В результате автоматического создания СПЗ обработкой Формирование строк плана закупок создаются новые версии СПЗ содержащие ссылку на родительскую строку плана.
