Акцепт / Протест переводного векселя

Документ регистрирует факт акцепта (принятия обязательства оплатить задолженность) на Плательщика по переводному векселю. В случае несогласия принять обязательство, Плательщик направляет Векселедержателю протест.

Графическая схема справки

Взаимосвязь данного документа с объектами системы можно посмотреть на схеме:

  • Операции с векселями (со стороны Векселедателя)
  • Операции с векселями (со стороны Плательщика)
  • Операции с векселями (со стороны Векселедержателя)

Другие графические схемы справки можно посмотреть по ссылке.

Предъявление к акцепту переводного векселя

Операция Векселедержателя. Регистрирует исполнителя по погашению долга в результате акцепта векселя (принятие задолженности) Плательщиком. Если Плательщик подаст нотариально заверенный протест, то обязанность по уплате возвращается Векселедателю.

Вексель - элемент справочника Ценные бумаги, объект учета.
Должник - Плательщик, Организация, которой направляется вексель для акцепта.
Векселедержатель - Организация, предъявляющая вексель к акцепту.
Статус акцепта - содержит значения (Акцептован, Протест в неакцепте, Не акцептовался)
В момент, когда Векселедержатель регистрирует вексель документом Регистрация векселя ценная бумага может быть акцептована или не акцептована.
По предъявлению документом Акцепт / Протест переводного векселя с видом опереции Предъявление к акцепту переводного векселя присваивается статус Акцептован или Протест в неакцепте
Ответственный - пользователь, ответственный за выполнение операции.
Комментарий - строка, в которой пользователь может добавить произвольную информацию по операции.
Документ используется для отражения в учете векселедержателя операций по предъявлению векселя к акцепту плательщику.
В момент проведения документа программа формирует проводки по перемене лиц в обязательствах – задолженность векселедателя перевыставляется на плательщика по векселю.
В документе необходимо указать статус акцепта векселя (акцептован), указать договор акцепта и нового должника. Стоимость векселя в документе не указывается.
Если выпускаемый вексель является переводным, он может быть акцептован плательщиком уже в момент выпуска. Тогда в документе можно установить флаг «Акцептован», в результате будут сформированы бухгалтерские проводки по акцепту векселя. Факт акцепта векселя и соответствующие проводки могут также быть сформированы независимо от выпуска векселя, если он акцептован позже. Для этого необходимо провести документ «Уведомление об акцепте».
Вексель может быть выпущен против платежа или против задолженности (необходимо указать в документе «Выпуск простого/переводного векселя). Если вексель выпускается против задолженности, в качестве счета учета финансового инструмента необходимо указать счет товарного векселя 60.3 или 62.3. Вексель, выпускаемый против платежа является денежным и может учитываться на счете финансовых вложений (58).
В момент проведения документа формируются следующие проводки:
Дебет Счет взаиморасчетов с плательщиком
 Кредит Счет учета финансового инструмента
перенос стоимости векселя на плательщика (изменение лиц в обязательствах)
Дебет Счет взаиморасчетов с плательщиком
 Кредит Счет учета премии/дисконта
перенос дисконта по векселю на плательщика (изменение лиц в обязательствах)

 

Уведомление об акцепте переводного векселя

Операция Векселедержателя. Регистрирует факт акцепта векселя (принятие задолженности) Плательщиком т.е. переход долга с Векселедателя на Плательщика.

Вексель - элемент справочника Ценные бумаги, объект учета.
Акцептант - Плательщик, Организация, акцептующая вексель.
Векселедатель - Организация, выпустившая вексель.
Должник - Организация, векселедатель, выпустившая вексель.
Договор акцепта - элемент справочника Договоры. 
Сумма операции - сумма, которую признает акцептант. В частном случае, может не совпадать с суммой предъявляемой к Плательщику.
Объект расчетов - информация об объекте расчетов. Таким объектом в системе может выступать Договор / Ценная бумага / Заказ поставщику)
Ответственный - пользователь, ответственный за выполнение операции.
Комментарий - строка, в которой пользователь может добавить произвольную информацию по операции.

Документ используется для отражения в учете векселедателя операций по акцепту векселя плательщиком.
Если акцепт векселя плательщиком происходит одновременно с выпуском векселя, данный документ можно не использовать, вместо этого установить флаг «Акцептован» в документе «Выпуск переводного векселя».
Сумма акцепта в документе не указывается, документ предназначен для полного акцепта векселя (по номиналу), необходимо указать договор акцепта векслея.
В момент проведения документа программа формирует проводки по перемене лиц в обязательствах – задолженность плательщика векселедателю перевыставляется на векселедержателя, а также по признаню расходов в виде дисконта по акцептованному векселю.
В момент проведения документа формируются следующие проводки:
Дебет Счет учета финансового инструмента
 Кредит Счет взаиморасчетов с плательщиком
зачет задолженности плательщика при акцепте векселя плательщиком (на сумму номинала векселя)
Дебет Счет начисления процентов
 Кредит Счет РБП
признание расходов в виде дисконта по векселю при акцепте

Уведомление о протесте переводного векселя

Операция Векселедержателя. Регистрирует факт отказа от акцепта векселя (принятие задолженности) Плательщиком. При нотариальном заверении Протеста обязанность по оплате переходит к Векселедателю. Данная операция регистрирует переход долга с Плательщика на Векселедателя.

Вексель - элемент справочника Ценные бумаги, объект учета.
Векселедатель - Организация, выпустившая вексель.
Должник - Организация, записанная в векселе как Плательщик (именно данная организация подает протест, а регистрирует протест Векселедатель).
Векселедержатель - Организация, предъявившая вексель к акцепту.
Сумма операции - сумма, которую не признает акцептант.
Начисленный расход - начисленный на момент протеста расход по векселю с учетом процентов, если условия, прописанные в векселе, предусматривают начисление процентов.
Ожидаемый расход - Расчет системы для отражения в плановых графиках.
Открыть график начисления - гиперссылка на плановый график по векселю.
Ответственный - пользователь, ответственный за выполнение операции.
Комментарий - строка, в которой пользователь может добавить произвольную информацию по операции.

Документ используется для фикасации факта протеста векселя плательщиком в учете векселедателя. Данная операция не требует отражения в учете векселедателя, поскольку не вызвает никаких изменений в обязательствах.

Акцепт переводных векселей

Регистрирует факт акцепта векселя (принятие задолженности) Плательщиком. Данную операцию роегистрирует Плательщик.

Вексель - элемент справочника Ценные бумаги, объект учета.
Акцептант - Плательщик, Организация, акцептующая вексель.
Векселедержатель - Организация, предъявившая вексель к акцепту
Параметры векселя - основные данные по векселю (из справочник Ценные бумаги)
Векселедатель - Организация, выпустившая вексель.
Договор акцепта - элемент справчоника Договоры. 
Объект расчетов - информация о объекти расчетов. Таким объектом в системе может выступать Договор / Ценная бумага / Заказ поставщику)
Сумма акцепта - сумма, которую признает акцептант. В частном случае, может не совпадать с суммой предъявляемой к Плательщику.
Ожидаемый расход - Расчет системы для отражения в плановых графиках, выполненный с допущением, что вексель будет погашен в сроки, указанные в параметрах векселя. Реальный расход будет зафиксирован при предъявлении векселя к погашению.
Сумма платежа - Текущая стоимость плюс ожидаемый расход. Сумма платежа, расчитанная системой.
Доходность - % годовых по векселю. Процентный доход применяется только для векселей по предъявлению и начисляется с даты возможного предъявления. Дата, с которой начинается начисление процентов можут быть указана явно.
Открыть график начисления - гиперссылка на плановый график по векселю.
Ответственный - пользователь, отвественный за выполнение операции.
Комментарий - строка, в которой пользователь может добавить произвольную информацию по операции.
Документ используется для отражения в учете плательщика операций по переводу долга плательщика векселедателю на векселедержателя.
Сумма акцепта в документе не указывается, документ предназначен для полного акцепта векселя (по номиналу).
В момент проведения документа программа формирует проводки по перемене лиц в обязательствах – задолженность плательщика векселедателю перевыставляется на векселедержателя.
В настройке счетов учета необходимо указать счета:
Счет учета финансового инструмента  – указывается счет, на котором в учете плательщика будет числиться задолженность плательщика перед новым векселедержателем
Счет взаиморасчетов с плательщиком – указывается счет, на котором в учете плательщика числиться задолженность перед векселедателем, которая погашается в результате акцепта векселя плательщиком.
В момент проведения документа формируются следующие проводки:
Дебет Счет взаиморасчетов с плательщиком
 Кредит Счет учета финансового инструмента
зачет задолженности векселедателя при акцепте векселя плательщиком (на сумму номинала векселя)
