Версия соглашения депозит

Документ предназначен для задания настроек финансового договора по размещению средств.  Документ обслуживает виды договоров по финансовым инструментам: депозит, заем выданный, договора о минимальном неснижаемом остатке (договора МНО). 

Графическая схема справки

Взаимосвязь данного документа с объектами системы можно посмотреть на схеме.

Кнопки командной панели

  • Комментировать документ - открывает форму ввода произвольного комментария по параметрам финансового соглашения, а также приложить файлы, например график расчетов.
  • Движения документа - выход на записи регистров по соглашению.
  • Редактировать - кнопка меню, позволяет создать новую версию или отредактировать существующую.
  • Вычисления - кнопка меню, позволяет задать правило пересчета сумм при работе с графиком:
    • Автоматически - после каждого изменения выполняется пересчет связанной информации.
    • Вручную - пересчет выполняется по команде (кнопке) пользователя Пересчитать график.
  • Отчеты - кнопка меню, позволяющая открыть отчеты данного соглашения.
    • Сводный график - отчет выводит в одной форме данные рамочного соглашения и отдельных соглашений. Отчет наиболее актуален для вывода графика расчетов по рамочному соглашения.   
    • Анализ графика - отчет с двумя вариантами вывода данных: по динамике изменения задолженности, по структуре задолженности.
  • Открыть - кнопка меню, позволяет открыть действующий график, опорный график или исходный график.

    • Соглашение может содержать значительное количество версий графика. Например, график может содержать версию графика согласованную с банком , версию графика после пополнения депозита или частичного досрочного возврата, версия прогноза по операции пополнения или досрочного возврата, редактируемые версии, а также последняя актуальная.

      • Исходный график - первоначальная версия графика, сохраняемая в системе. Используется для анализа отклонений.
      • Опорный график - одна из достоверных версий графика, согласованнная с банком.
      • Действующий график - последняя рабочая версия графика.  

    Управление версиями выполняется на вкладке Расчеты формы договора по команде Все версии графика (кнопка меню См.также).

    Основное

  • Дата – дата оформления карточки инструмента.
  • Опорный график – при установленном флажке график назначается опорным.
  • Сумма – сумма по договору / спецификации. 
  • Валюта –  валюта договора.
  • Период действия – период действия договора / спецификации.

    • Для депозита доступны реквизиты:

  • Пополняемый - признак, разрешающий довнесение средств на депозит.
  • Возможность досрочного возврата - признак, разрешающий частично раньше срока забрать средства на условиях оговоренных соглашением. При установке флажка доступен реквизит Ставка при досрочном возврате на закладке Проценты.
  • Возможность частичного дострочного возврата - признак, разрешающий раньше срока забрать часть средств на условиях оговоренных соглашением.

    • Для рамочного договора выводится группа реквизитов Ограничения, определяющая правила на отдельные спецификации в рамках рамочного договора. Данные реквизиты содержат информацию:

  • Сроки размещения не менее / не более;
  • Сумма размещения не менее / не более.

    • Ниже располагается общая информация Всего по договору (на dd.mm.yyyy): Получено, Доступно, Основной долг. 

    Проценты

  • Получение процентов  - параметр, влияющий на срок и периодичность получения процентов по депозиту;
  • Периодически - получение процентов производится с указанной периодичностью.
  • В конце срока – получение процентов производится один раз за весь период.
  • Длительность периода – периодичность (день, неделя, месяц, квартал итд). Реквизит доступен при установке переключателя в Периодически.
  • Вести отсчет от – дата, от которой начинается расчет процентов (даты открытия депозита, даты первой выборки, даты первого погашения). 
  • Уплата через [ ] дней от  [даты окончания периода, даты начала периода] – правило расчета дня уплаты процентов по наступлению срока. Реквизит доступен при установке переключателя в Периодически.
  • Так же получать при погашениях тела – при установленном флажке при возврате депозита досрочно уплачиваются начисленные проценты на дату операции, расчитанные согласно ставке досрочного востребования депозита.
  • Правило переноса – правило переноса получения процентов, при попадании расчетного дня выплатыпроцентов на выходной или праздничный день. Доступны следующие значения: 
    Не переносить – не переносить, то есть зачисление процентов производится в расчетный день независимо от типа дня (выходной или праздничный) и календарного периода.
    Следующий рабочий день –  зачисление процентов переходит на ближайший рабочий день, следующий за расчетной датой. Например, если расчетный день выходной (праздничный), то зачисление процентов будет выполнено в первый день, после выходных (праздников).
    Предыдущий рабочий день – зачисление процентов переходит на ближайший рабочий день, предшествующий расчетной дате.  Например, если расчетный день выходной (праздничный), то зачисление процентов должно быть выполнено не позднее последнего рабочего дня, предшествующего выходному (праздничному) дню.
    Следующий рабочий день отчетного периода – зачисление процентов переходит на следующий рабочий день, только в том случае, если этот рабочий день принадлежит тому же календарному периоду, что и расчетная дата. Например, период март. Если расчетный день уплаты процентов попадает на выходной, и ближайший рабочий день в марте, то зачисление процентов должно быть в ближайший рабочий день, следующий за расчетной  датой. Иначе, зачисление процентов должно быть выполнено в последний рабочий день, предшествующий расчетной дате.
    Предыдущий рабочий день отчетного периода – зачисление процентов переходит на предыдущий рабочий день, только в том случае, если этот рабочий день принадлежит тому же календарному периоду, что и расчетная дата. Например,  период март. Если расчетный день уплаты зачисления процентов попадает на выходной, и предшествующий расчетной дате рабочий день попадает в март, то зачисление процентов должно быть в  последний рабочий день, предшествующей расчетной  дате. Иначе, зачисление процентов переходит на ближайший рабочий день, следующего месяца.
    С изменением процентного периода – признак, определяющий правило расчета процентов при попадании расчетной даты на выходной (праздничный день). Расчет процентов выполняется в календарных днях без учета праздничных и выходных дней. Если флажок включен, то количество календарных дней, принимаемых в расчет, будет следовать правилу переноса даты зачисления процентов. Если флажок выключен, то независимо от заданного правила переноса даты уплаты процентов количество календарных дней считается от расчетной даты до расчетной даты.

    • Процентная ставка по займам выданным

    Для займов возможно указание фиксированных и плавающих кредитных ставок, а так же ручной ввод значения ставок.

    Фиксированная процентная ставка – ставка может быть зафиксирована договором заимствования на весь период, независимо от внешних факторов, а может изменяться со временем. При этом изменения вступают в силу с четко определенного договором периода. Ставку, изменяющуюся на фиксированное значение в течение срока договора, часто называют дифференцированной.

    Для установки нужной ставки необходимо включить  комбинацию переключателей. Возможны комбинации:
    Фиксированная ставка + Ставка неизменна  - явно указанная процентная ставка действует все время действия договора. Необходимо заполнить реквизит Процентная ставка.
    Фиксированная ставка + Ставка изменяется во времени – явно указанная ставка действует в явно заданном временном интервале, после чего вступает в силу другая явно указанная ставка. При выборе данной комбинации необходимо пройти по гиперссылке и явно установить перечень ставок и дат, с которых ставки вступают в силу.
     
    Плавающая процентная ставка – ставка, изменяемая во времени, база расчета которой является плавающая, заранее не известная величина. Основой базы расчета плавающих ставок являются различные котировки со смещением на оговоренный договором процент и ограничения по различным условиям на расчет котировок. 
     
    Процентная ставка + смещение [  ]% годовых – база для расчета процентной ставки, которая может быть увеличена  на явно указанное значение процента. Базой расчета плавающей ставки могут быть различные котировки:
  • Курсы валют – основой расчета являются курсы валют.
  • Рыночная ставка процента – основой расчета являются ведущие индикаторы (например, MosPrime Rate, libor, euribor);
  • Банковская котировка – основой расчета являются финансовые показатели конкретного банка. За основу могут быть взяты любые показатели, например курс конкретной валюты в указанном банке;
  • Банковская процентная ставка – основой расчета являются процентные ставки конкретного банка.
    • Не менее [  ]%, не более [  ]% - ограничения на результаты расчета  процентной ставки. Полученная в результате процентная ставка проверяется на граничные условия [не менее и не более] указанной величины. В случае выхода за границы ограничений берется ближайшее  граничное значение.

    Ставка фиксируется за N дней – правило, по которому процентная ставка фиксируется за указанное количество дней от даты начала периода или даты окончания периода.
    Ручной ввод значений ставок – предоставляет Пользователю доступ к ручному вводу плавающих процентных ставок. Пройдя по гиперссылке, пользователь может ввести значения плавающей процентной ставки вручную.

    Расчетная база -  параметр, по которому считаются количество дней процентного периода. Доступны для выбора следующие предопределенные значения:

  • Год: 360 дней, месяц: 30 дней – количество дней в году и месяце четко задано указанными значениями.
  • Год: 360 дней, месяц: по факту – количество дней в году четко задано, количество дней рассчитывается по календарю.
  • Год: по факту, месяц: 30 дней – количество дней в году рассчитывается по календарю, количество дней в месяце четко задано.
  • Год: по факту, месяц по факту – количество дней года и месяца определяется календарем.

    • Процентная ставка по депозитам

    Для депозитов возможно указание только фиксированных ставок.

  • Процентная ставка - фиксированная процентная ставка по депозиту.
  • Капитализация процентов - признак, указывающий, что по депозиту, при пополнении в периоде размещения, будет выполненв капитализация процентов. Доступен только для  депозитов с включенным признаком Возможно пополнение.
  • Ставка при досрочном востребовании - фиксированная ставка, применяемая при частичном или полном востребовании депозита раньше срока.
  • Метод зачисления – метод зачета размещенного депозита LIFO /FIFO, в случае пополняемого депозита.
    • Расчетная база задана для депозитов задана жестко. Расчет выполняетмя с параметром Год: 360 дней, месяц: 30 дней – количество дней в году и месяце четко задано указанными значениями.

    График расчетов

    На закладке график выводятся результаты расчета графика начислений и оплат по финансовому инструменту. График хранится по версиям. Система регистрирует изменения, вносимые в график и сохраняет новые данные. Для работы с версиями на панель выведена кнопка меню Редактировать.

  • Отображать - переключатель режима работы с  данными графика.
  • Текущее значение – отображает действующий график. Набор колонок зависит от переключателей (Начисления, Комиссии и штрафы, Поправки)
  • Изменения – отображает график в режиме изменения. В данном режиме рядом с кнопкой меню Отображать можно определить версию графика, относительно которой будет выполнено сравнение.
  • Вывести список - вывод графика для копирования или печати в табличном или текстовом виде.
  • Найти - поиск строки графика. Отмена поиска выполняется одноименной кнопкой командного интерфейса.  
  • Загрузить график - позволяет загрузить данные графика путем копирования через буфер обмена. Для загрузки графика разработана форма загрузки.
  • Обновить фактические данные - по каждому договору проверяется наличие фактических движений. Данные отображаются в графике серым цветом.
  • Пересчитать график - пересчитывает связанные ячейки графика. Используется при ручном режиме пересчета при работе с графиком.
  • Добавить - позволяет добавить новые строки графика в режиме редактирования или создания новой версии.

    • В строках графика выводится информация с точностью до даты планируемой или фактической операции. В колонках данные представлены с группировкой по основному долгу, процентам, штрафам и комиссиям. Дополнительно можно вывести данные по начислениям. При работе с графиком работает интерактивная расшифровка. Каждой ячейке с данными доступна расшифровка значения вызываемая правой кнопкой мыши по команде Показать историю изменений. Изменения показываются в режиме изменения версий значения. Опорный график явно обозначен надписью (опорный).

    График выводится по видам соглашения Договору с условием и Спецификация к договору рамочного договора. Сводный график по рамочному договору (со всеми спецификациями) можно посмотреть отчетом Сводный график.

    Исполнение графика расчетов

    Работа с графиками по исполнению выполняется из обработки Исполнение финансовых договоров. Обработка предоставляет удобное рабочее место для менеджера финансового договора, предоставляя инструменты для корректировки графиков, отслеживания просроченных платежей, создания заявок на оплату. Подробнее смотри справку к обработке Исполнение финансовых договоров.
