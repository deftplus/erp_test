Операционный план

Оперативное планирование применяется для отражения конкретных операций, которые с высокой степенью достоверности будут совершены. Данные оперативных планов можно использовать для прогноза изменения остатков на денежных счетах, состояния взаиморасчетов, доступности необходимых ресурсов, а также контролировать получение различных выгод (например, централизованно управлять закупками). Если в организации ведется бюджетирование, то оперативное планирование его эффективно дополняет, реализуя функции резервирования и контроля конкретной операции. При отсутствии регламентированного бюджетного процесса операционное планирование может выступать «легким» вариантом бюджетирования, обеспечивая надежный контроль за исполнением операций и план-фактный анализ отклонений. 

 

Графическая схема справки 

- Установка и контроль лимитов
- Казначейство
- Корпоративные закупки
- Подготовка и проведение торгов 
С другими схемами системы можно ознакомиться по ссылке.

Основная информация 


В основе операционного планирования лежат три вида бюджетов:

  • Бюджет движения денежных средств;
  • бюджет доходов и расходов;
  • бюджет закупок.

Задачи операционного планирования решаются с помощью документов: 

  • Операционный план,
  • Заявка на оплату,
  • Заявка на расход,
  • Ожидаемое поступление денежных средств,
  • Планируемый доход,
  • Договор (Версия соглашения коммерческого договора),
  • Заявка на корректировку лимитов,
  • Корректировка лимитов,
  • Отражение фактических данных.

Связь операционного планирования с бюджетированием поддерживается по видам бюджетов и аналитикам, задаваемым при настройке вида оперативного плана. Структура и внешний вид плана определяются настройками "Планирование и лимитирование" ("Структура плана", "Варианты настроек") раздела "Планирование и контроль".

Обратите внимание, что при выключенном переключателе "Использовать" в ("Параметры планирование", "Параметры лимитирование") пользователю будут недоступны заявки операционного планирования по виду бюджета. Использование заявок на корректировку задается одноименным переключателем на форме "Планирование и лимитирование". 


Виды операции документа


Установка и корректировка лимитов

Для установки и корректировки лимитов используется вид операции "Лимиты". Операционный план всегда пишет в регистры лимитирования только корректирующие значения. таким образом, если лимиты устанавливаются бюджетами, операционные план подгружает установленные лимиты и позволяет их скорректировать. Если бюджетирование не используется или лимиты устанавливаются только операционными планам, то установленные значения будут отражаться в колонке отчета по лимитам "Корректировка", что нужно считать особенностью системы. 


Планирование и корректировка плана

Для операций планирования используется вид операции "Планирование". Операция для ввода и корректировки планов одна. Пользователю достаточно заполнить шапку документа по выбранной настройке плана и перейти на закладку "Аналитики планирования". По заполненным аналитикам шапки документа из регистра планирования подтянутся данные в табличную часть. Если данные в регистре есть, то документ автоматически переходит в режим корректировки. В командной панели формы доступны кнопки "Новое" , "Корректировка", "Исходное". Для удобства работы с табличной частью команды оснащены кнопками отображения "Скрыть неизмененные колонки", "Показать количество", "Показать сумму". При работе с планом вида бюджета закупок добавляются кнопки ускоряющие ввод данных "Тип номенклатуры" (позволяет зафиксировать тип "Номенклатура" или "Товарная категория" чтобы пользователь не выбирал на повторяющихся операциях одно и тоже значение составного типа), "Исходная номенклатура", "Заменить номенклатуру" (замена на аналоги выполняется для сокращения затрат на закупку). при необходимости скорректировать план можно использовать ввод на основании, копирование плана (F9) или создание и заполнение аналитик шапки вручную. результат будет одинаковым. Старая версия плана всегда будет недоступна для редактирования, а для корректировки создается новый документ с аналитиками шапки документа. Такая архитектура позволяет всегда отследить в отчетах выполненные изменения.  


Резервирование и корректировка резерва

Для операций планирования используется вид операции "Резервирование". Операция включается переключателем параметрах лимитирования "Использовать резервирование". Пользователю дополнительно нужно определить контроль при нарушении с операциями резервирования "Информировать", "Блокировать проведение документа", "Не контролировать". Устанавливает и корректирует резерв только операционный план, а сам документ выступает источником резерва для операций с резервами и обязателен для списания резерва.
