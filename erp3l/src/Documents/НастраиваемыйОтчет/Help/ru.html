Документ "Экземпляр отчета"

Документ "Экземпляр отчета" предназначен для сохранения в информационной базе информации сформированных отчетов: значений показателей и  параметров.

Форма списка документов "Экземпляр отчета", с одной стороны, позволяет формировать новые отчеты, с другой стороны, позволяет просматривать на экране компьютера ранее сформированные отчеты, исправлять их и распечатывать.

При формировании нового экземпляра отчета прежде всего необходимо указать вид отчета и бланк отчета. Остальные параметры отчета могут заполняться как при формировании экземпляра, так и в процессе его редактирования при нажатии кнопки «».

Параметры «Организационная единица», «Сценарий», «Период», «Основная валюта» и реквизит «Правила обработки» к моменту сохранения отчета обязательно должны быть заполнены. Необходимость заполнения остальных реквизитов определяется содержанием, способами заполнения и проверки отчета:

  • Бланк импорта – бланк, на котором заполнен внешний файл, который импортируется в ИБ;
  • Правило проверки;
  • Используемая ИБ – внешняя ИБ, данные которой используются для заполнения показателей;
  • Единица измерения – финансовые показатели экземпляра отчета могут измеряться в единицах, тысячах или миллионах;
  • Точность – количество знаков после запятой;
  • Исполнитель – пользователь, заполняющий (импортирующий) отчет;
  • Проверяющий – пользователь, утверждающий отчет

В конфигурации предусмотрено автоматическое назначение перечисленных реквизитов документов "Экземпляр отчета" при запуске рабочего процесса.

Дополнительные валюты отчета могут быть указаны на закладке «Дополнительно». Для использования дополнительных валют необходимо выполнение условий:

  • Для финансовых показателей отчета должен быть задан вид курса;
  • Должны быть заданы курсы всех используемых валют.

Основная форма документа предназначена для отображения сформированного отчета на экране компьютера, заполнения и редактирования его показателей. Представление отчета определяется бланком, выбранным при его формировании. Выбрать другой бланк отчета можно в нижней части формы в поле «Бланк отчета».

Значения заполняемых показателей могут быть определены автоматически или введены (изменены) вручную. Автоматическое заполнение осуществляется при нажатии кнопки «Заполнить». Предусмотрены следующие режимы автоматического заполнения:

  • По правилу обработки – выполнение процедур и расчет по формулам, определенным правилом обработки. Нажатие кнопки "Включить расширенную диагностику расчета показателей" обеспечивает включение режима расширенной диагностики ошибок и предупреждений, возникающих при заполнении по правилу обработки.
  •  Импорт из файла – импорт значений показателей из внешнего файла формата MXL или MS Excel. Файл может быть внешним (режим «На диске») или предварительно сохраненным в информационной базе в справочнике "Сохраненные файлы организационных единиц". При импорте предусмотрен ряд проверок данных. Если процедура импорта выявит несоответствие данных бланку отчета, то на экран будет выведено окно сопоставления бланка и импортируемого (исходного файла) с выделенными областями расхождений". Если определенной в настройке субконто комбинации реквизитов аналитики, определяющей ключ при импорте, соответствует несколько записей в соответствующем справочнике, то при импорте групп раскрытия, содержащих эту аналитику, пользователю предлагается выбрать необходимую запись.
  • Свернуть по аналитикам отчетов - заполнение экземпляра отчета аналитики путем суммирования данных нескольких отчетов, которые уже записаны в информационную базу. Сворачивание по аналитикам возможно только для отчетов, для которых указаны виды аналитики, причем для заполняемого экземпляра отчета значение аналитик не должно быть заполнено: суммируются значения показателей отчетов с теми же параметрами и с заполненными значениями аналитик.
  • Свернуть по периоду – заполнение экземпляра отчета за период путем суммирования данных отчетов за входящие в него периоды.
  • Консолидировать периметр – консолидация данных отчетов организаций, включенных в периметр, за тот же период, по тому же сценарию и в той же валюте. Опция доступна только при формировании отчетов консолидирующих организаций. Метод консолидации для каждой из организаций, включенной в периметр, определен в справочнике «Состав периметра», там же определен индекс присутствия каждой организации в периоде. В полном объеме учитываются показатели организаций, для которых предусмотрена полная консолидация; показатели организаций, для которых предусмотрено долевое участие, не учитываются. Индекс присутствия организаций в периоде учитывается независимо от метода консолидации, но только для тех показателей отчета, для которых заполнен флаг «Учитывать индекс присутствия в периметре при консолидации».

Если при импорте данных из MS Excel возникает ошибка открытия файла, то рекомендуется выполнить следующее:
Для 64-битных операционных систем
Сделать папку %WINDIR%\config\systemprofile\Desktop
Для 32-битных операционных систем
Сделать папку %WINDIR%\System32\config\systemprofile\Desktop

При заполнении показателей нажатие кнопки "Использовать предварительные данные" обеспечивает использование значений показателей других отчетов текущей ИБ, независимо от их состояния. При отжатой кнопке используются только показатели отчетов в состоянии "Подготовлен" и «Утвержден».

Для возможности ручной корректировки показателей отчетов нужно нажать кнопку «Перейти в режим редактирования отчета». Поддерживается ручная корректировка значений показателей непосредственно в форме документа с сохранением истории корректировок. Для использования этого способа необходимо включить режим версионирования данных отчетов (меню "Операции" — "Константы").

Значения вычисляемых показателей всегда рассчитываются автоматически.  Такой расчет может производиться интерактивно, если в меню кнопки   "Вычислить" перевести кнопку "Вычислять при изменении" в нажатое состояние. Для запуска перерасчета после завершения ручных правок нужно в меню кнопки "Вычислить" выбрать пункт "Вычислить сейчас".

Правильность заполнения экземпляра отчета осуществляется при нажатии кнопки «Проверить»: запускается процедура проверки и проверяются контрольные соотношения, определяемые правилами проверки.

Кнопка "Очистить" позволяет очистить значения показателей.

Кнопка   «Операции с показателем» предусматривает следующие операции:

  • Раскрытие - вызывает на экран форму "Раскрытие показателей", содержащую перечень значений субконто, из которых складывается значение выделенного показателя;
  • Расшифровать – выводит на экран формулу расчета или заполнения показателя, значения операндов формулы;
  • Ввод ручной операции – формирует документ «Операции» с параметрами, совпадающими с параметрами экземпляра отчета;
  • Изменить и распределить корректировку – производит внесение поправок в консолидированный отчет и их обратное распределение по исходным отчетам
  • Заполнить по образцу – заполняет значения показателей с учетом данных других экземпляров отчетов, причем способ заполнения указывается для конкретного экземпляра отчета непосредственно при его заполнении.

 

Кнопка  "Выделить раскрываемые ячейки" выделяет все ячейки, соответствующие раскрываемым показателям в макете.

Кнопки «Скрыть пустые строки» и «Скрыть пустые колонки» позволяют представить отчет в более компактном виде — без вывода соответственно пустых строк и пустых колонок.

 

Кнопка "Запретить вносить изменения в отчет при обработки его в управлении процессом" позволяет блокировать экземпляр отчета от изменений, автоматически формируемых в ходе выполнения процесса (например, от автоматического заполнения отчета).

 

Степень готовности экземпляра отчета определяется реквизитом «Состояние», расположенным в нижней части формы документа. Способы изменения состояния зависят от состояние, в которое должен быть переведен отчет, и от способа согласования, определенного этапом и матрицей полномочий процесса, в рамках которого формируется отчет. Для экземпляров отчетов, для которых не определен маршрут согласования, все процесс вручную переводится в каждое состояние, включая состояние "Утвержден". Согласование экземпляра в рамках маршрута согласования начинается с того, что исполнитель отчета вручную переводит его в состояние "Подготовлен". После этого состояние отчета не может быть изменено вручную ни одним из участников процесса. Утверждение или отклонение отчета производится пользователем, включенным в маршрут согласования, в форме "Согласование", после выполнения команды "Согласовать" контекстного меню кнопки "Согласование отчета". Эта кнопка видна из формы экземпляра отчета только тем пользователям, которые включены в этап согласования.

 

Если для отчета определены основная и дополнительные валюты, ввод и просмотр его показателей может осуществляться в каждой из этих валют. Выбор валюты осуществляется в поле «Отображать в валюте», расположенном в нижней части формы документа.
