Аккредитация поставщика

Предназначен для проведения аккредитации поставщика. Фиксирует требования к поставщику, критерии и документы, подтверждающие квалификацию поставщика. В результате проверки данных поставщика в документе фиксируется решение по процедуре и обоснование принятого решения. 

Графическая схема справки 

- Корпоративные закупки
- Аккредитация поставщиков 
- Подготовка и проведение закупок 
С другими схемами системы можно ознакомиться по ссылке.
 

Заполнение документа

В шапке документа выводится срок аккредитации поставщика. Срок рассчитывается автоматически от константы Срок аккредитации по умолчанию. Проверяющий имеет возможность уточнить дату. 

Анкета поставщика содержит ссылку на данные поставщика. Рядом выводится статус поставщика. 

Решение по документу - перечисление, содержит три значения: положительное решение, отказ, решение не принято. При проверке документ проверяется значение поля решение по каждой строке табличной части, но на смену статуса поставщика влияет только данный реквизит. 

Табличная часть Общие требования содержит требования аккредитации, критерии, подтверждающие документы, ответ, комментарий участника. Проверяющему дополнительно выводятся колонки: решение положительно, комментарий комиссии. При вводе ответа допустимо три значения (выбор значения назначается в настройках требования): ввод строкового значения (текст), выбор из заданного комиссией перечня значений, ввод числового значения в заданном диапазоне. 

Табличная часть Номенклатура поставщика содержит группы справочника номенклатура. Фиксирует номенклатуру, которую поставщик обязуется поставлять. На этапе квалификационного отбора включение номенклатуры в табличную часть лота взводит флажок Требуется квалификационный отбор. На решение о аккредитации поставщика данные закладки не влияют т.к. поставщик проходит аккредитацию в целом по организации. 

Табличная часть Комиссия по аккредитации содержит информацию о участниках комиссии, проводивших проверку документов Поставщика и ответственных за принимаемое решение. Табличная часть доступна только проверяющему т.е. пользователю системы и не доступно внешнему пользователю.

Табличная часть Товарные категории. Отображается на форме, если в настройках включена константа "Разделять номенклатуру по товарным категориям". Поставщик не знает наименования номенклатуры организации заказчика и тем более название товарной категории продукции которую он планирует поставлять. Данные в таблицу заносит представитель Заказчика, назначая поставщику товарные категории продукции, которую тот планирует поставлять. Может использоваться в закупочной процедуре для приглашения аккредитованных поставщиков, поставляющих номенклатуру, объявленную к закупке. 

Документ может быть введен из личного кабинета поставщика самостоятельно или на основании Анкеты поставщика. При отправке документа на проверку из личного кабинета поставщика фиксируется дата и время отправки. Данные используются в отчете Состояние аккредитации, вариант Аккредитация на рассмотрении. Отчет использует константу Длительность рассмотрения заявки на аккредитацию, рассчитывает время нахождения документа на проверке.
