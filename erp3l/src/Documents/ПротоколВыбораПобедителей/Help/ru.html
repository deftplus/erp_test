Документ "Протокол выбора победителей"

Содержит функционал по выбору поставщика и регистрирует результат работы закупочной комиссии. Оценка предложений участников выполняется только в протоколе. Протокол позволяет зафиксировать следующие решения:

  • Закупка состоялась, победитель определен.
  • Переторжка (указывается причина принятого решения, может проводиться несколько раз).
  • Закупка не состоялась (конкурентная закупка), но один поставщик, соответствующий требованиям прошел процедуру. Принято решение заключить с ним договор, чтобы не тратить время и ресурсы на проведение очередной процедуры. Такие закупки проходят отдельной позицией в отчетах 223-ФЗ.
  • Закупка не состоялась (указывается причина). Возможно, будет инициирована новая закупочная процедура.

Заполнение документа

Протокол обязательно содержит номер, дату подписания и принятое решение. Эта информация доступна пользователю на первой закладке. На закладке "Предложения участников" проводится работа по выявлению победителей.  Независимо от метода оценки выход на оценку выполняется командой "Рассмотрение предложений участников". Количество шагов оценки зависит от вида закупки (устанавливается в лоте). 

Лотовая закупка

При оценке по критериям (бальная оценка) проходит в два шага. На первом эксперты оценивают предложения, на втором члены комиссии фиксируют победителя. Данный вид закупки может иметь только одного победителя. 

При оценке по ценам проходит в один шаг. Все предложения ранжированы по возрастанию цены, члены комиссии только фиксируют победителя. Данный вид закупки может иметь только одного победителя.

Попозиционная закупка

При оценке по критериям (бальная оценка) проходит в два или три шага. На первом эксперты оценивают предложения, на втором члены комиссии просматривают и отбирают предложения с наивысшими баллами. Могут зафиксировать победителя, а могут третьим шагом распределить объем закупки между несколькими участниками. Данный вид закупки может иметь несколько победителей (у каждой позиции может быть свой поставщик). 

При оценке по ценам проходит в один или два шага. Все предложения уже ранжированы по возрастанию цены, члены комиссии могут зафиксировать победителя по каждой позиции лота, а могут вторым шагом перераспределить объем между участниками. Данный вид закупки может иметь несколько победителей.

Зонтичная закупка

При оценке по критериям (бальная оценка) проходит в два шага. На первом эксперты оценивают предложения, на втором члены комиссии просматривают и отбирают предложения с наивысшими баллами и фиксируют победителей. Объем закупки автоматически распределяется между участниками по заданному в лоте условиям зонтичной закупки (объем договора зависит от места участника). Данный вид закупки имеет четко фиксированное (в лоте) число победителей. 

При оценке по ценам проходит в один шаг. Все предложения уже ранжированы по возрастанию цены, члены комиссии только фиксируют победителей по каждой позиции лота. Накладывается шаблон распределения заданный в лоте (условия зонтичной закупки). Данный вид закупки имеет четко фиксированное (в лоте) число победителей. 

Переторжка

Если в закупочной процедуре включен флажок "Доступна переторжка", то в протоколе можно принять решение о переторжке (переключатель "Переторжка"). После установки переключателя в значения "Переторжка" на форме открывается табличная часть для отбора участников переторжки из списка участников закупочной процедуры. По кнопке "Заполнить"в табличную часть добавляются все участники. Поданные предложения участников, на основании которых принято решение о переторжке сохраняются в протоколе. После проведения протокол блокируется от изменений. В системе регистрируют улучшенные предложения участников вводя новые предложения. Если участник не подал улучшенное предложение следует на основании предыдущего предложения создать новое (копированием). Это нужно сделать, чтобы все предложения попали в новую итерацию оценки предложений. Далее создают новый протокол, в котором проводят повторную оценку и принимают решения о результатах процедуры. Количество итераций переторжки не ограничено. Номер переторжки отображается на форме протокола и форме закупочной процедуры справа от флажка "Доступна переторжка". 

Заключение договора

После завершения процедуры по выбору победителей переходят к заключению договоров. Однако протокол должен быть надлежаще оформлен:

  • В протоколе прописана дата утверждения и принятое решение
  • Результат закупки "Состоялась успешно", "Не состоялась, но будет заключен договор с единственным поставщиком"
  • Протокол "Утвержден"
При соблюдении перечисленных условий на форме доступна команда "Управление договорами". В открывшейся форме договор создается по каждому победителю командой "Создать". В договор переходит из закупочной процедуры следующая информация:

  • График поставок - заполняется по спецификации протокола. В табличной части графика поставок для каждой позиции автоматически прописывается лот, номенклатура, количество, цена, сумма, ставка ндс, сумма ндс. 
  • Условия оплаты - заполняется из предложения победившего участника. 
  • Способ формирования платежей - "График" на закладке "Условия" договора.
  • График расчетов - формируется автоматически.
  • Ссылка на предложение победившего участника - на закладке "График поставок" добавляется гиперссылка на предложение участника для получеия информации о других условий поставщика (гарантийные обязательства, сроки поставки).
  • Данные закладки "223-ФЗ" - заполняются по данным закупочной процедуры для последующей выгрузки на ЕИС договора, сведений о договоре и отчетах по закупкам.

Размещение на ЕИС

Организации, работающие по 223-ФЗ выгружают протокол на ЕИС. Передавать следует только итоговый протокол по результату закупки ("Состоялась", "Не состоялась", "Не состоялась" [заключен договор с единственным поставщиком]). 

Размещение протокола на ЕИС выполняется обработкой "Обмена данными с ЕИС", закладка "Выгрузка". При первой выгрузке регистрационный номер размещения на ЕИС не заполняется. Проконтролируете заполнение поля УИД ЕИС (кнопка "Установить стандартный УИД"). После размещения протокола на ЕИС ему присваивается регистрационный номер, который можно установить вручную (скопировать c ЕИС и заполнить номер протоколе) или получить обработкой "Обмен данными с ЕИС", закладка "Загрузка". В случае изменения протокола необходимо открыть форму версий по кнопке "Версии ЕИС" и добавить в таблицу новую строку, указать причину внесения изменений. Будет создан новый идентификатор для регистрации изменений на ЭТП для повторной выгрузки.
