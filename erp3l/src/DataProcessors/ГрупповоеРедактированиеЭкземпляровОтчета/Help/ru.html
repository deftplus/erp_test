Обработка "Сводная таблица" позволяет обрабатывать данные нескольких экземпляров отчетов одного вида, имеющих различные ключевые реквизиты (период, сценарий, организационная единица, аналитика на уровне отчета) в многомерной таблице, по горизонтали которой развернуты расположены ключевые измерения и колонки отчетов, а по вертикали - ключевые измерения, строки отчетов и их аналитические раскрытия.

Иными словами редактируемый «многомерный куб» имеет следующие измерения:

  •  Период.
  • Организационная единица.
  • Сценарий.
  • Строки отчета.
  • Колонки отчета.
  • Аналитики на уровне отчета.

 

Инструмент удобен в первую очередь для бюджетирования и обработки управленческой отчетности. Поддерживается большинство операций, доступных в экземпляре отчета:

  • все виды расчета показателей;
  • комментирование показателей;
  • согласование отчетов;
  • работа с протоколами контрольных соотношений,;
  • обратное распределение показателей;
  • "планирование от достигнутого".

Инструмент имеет удобный навигатор, существенно упрощающий позиционирование и навигацию по массиву данных отчетности, образованному большим количеством отчетов.

 

Настройка сводной таблицы.

Определение массива редактируемых экземпляров отчетов производится на закладке "Настройка обработки". При этом указываются вид отчета, валюта представления и уточняются отборы, позволяющие отобрать экземпляры отчетов для обработки.

Доступны 3 варианта отбора обрабатываемых экземпляров отчетов:

  • отбор по ключевым измерениям (период, сценарий):одноименное значение переключателя;
  • отбор по списку экземпляров отчетов: одноименное значение переключателя; 
  • команда меню «Сводная таблица», доступного в форме списка экземпляров отчетов.

Имеется возможность указать бланк, используемый для отображения таблицы, который определяет отображаемые показатели и форматирование ячеек. Установленный флаг «Сохранять форматирование» применяет форматирование к ячейкам сводной таблицы, при этом незначительно замедляя ее отрисовку. Если флаг не установлен, используется содержащийся  в обработке макет по умолчанию, хранящий форматирование для 5 доступных уровней вывода многомерной таблицы.

В нижней части закладки «Настройка обработки» содержится панель, определяющая распределение измерений массива значений показателей по строкам и колонкам сводной таблицы.

При нажатии на кнопку «Получить данные» формируется локальный кэш данных. Если отсутствуют экземпляры отчетов, указанные в отборе, то при условии, что открыты соответствующие периоды, система формирует недостающие экземпляры отчетов. В качестве основной валюты формируемых отчетов используется валюта, указанная в поле «Основная валюта» панели настройки, в качестве дополнительных- оставшиеся.

При нажатии кнопки «Сформировать таблицу» обновляется визуальное представление сводной  таблицы в соответствии с настроенными параметрами.

Работа со сводной таблицей

Сводная таблица обеспечивает большинство операций, доступных в экземпляре отчета:

  • ручной ввод данных;
  • все виды расчета показателей;
  • комментирование показателей;
  • согласование отчетов;
  • работа с протоколами контрольных соотношений;
  • обратное распределение показателей;
  • "планирование от достигнутого".

Инструмент имеет удобный навигатор, существенно упрощающий позиционирование в массиве данных отчетности, образованном большим количеством экземпляров отчетов.

Панель инструментов и контекстные меню идентичны аналогичным элементам формы документа «Экземпляр отчета». Кроме этого, панель инструментов содержит дополнительную группу  кнопок для добавления, редактирования и удаления строк аналитических раскрытий показателей отчетов.

Для редактирования данных необходимо позиционироваться на соответствующей ячейке и перевести экземпляр отчета в режим редактирования. Блокируются данные только редактируемого экземпляра отчета.

Измененные показатели (включая вычисленные на основании измененных) раскрашиваются бледно-красным цветом, ячейки, недоступные для редактирования - бледно-серым. Запись в информационную базу данных, соответствующих редактируемому экземпляру отчета, производится при выключении кнопки «Редактирование экземпляра отчета». Также, все модифицированные данные могут быть записаны с помощью кнопки «Записать данные», расположенной на основной панели инструментов.

В правой части панели инструментов закладки «Редактирование» располагаются кнопки, обеспечивающие управление отображением сводной таблицы:

  • отображение/скрытие уровней иерархии строк;
  •  отображение/скрытие уровней иерархии колонок;
  • отображение/скрытие координат сводной таблицы;
  • отображение/скрытие панели навигатора, облегчающего позиционирование в массиве значений показателей.

Для отображения показателей сводной таблицы в дополнительных валютах используется (по аналогии с формой экземпляра отчета) поле со списком «Отображать в валюте:», расположенное в нижней части формы.

 

Использование навигатора

Панель навигатора предназначена для отображения координат, соответствующих текущему экземпляру отчета в сводной таблице и удобного позиционирования в сводной таблице.

Панель состоит из 4-х областей управления, соответствующих ключевым реквизитам на уровне экземпляра отчета:

·        организационные единицы;

·        периоды;

·        сценарии;

·        комбинации аналитик на уровне отчета, отображаемое в случае, если текущий вид отчета их имеет.

Каждая область управления включает табличное поле и кнопку включения/отключения отбора.

Табличное поле содержит колонки:

·        маркер, отображающий текущую позицию в массиве значений показателей;

·        наименование измерения;

·        флаг отбора по текущему значению ключевого измерения.

При перемещении курсора по сводной таблице марке в табличных полях позиционируется на текущее значение ключевого измерения.

При двойном клике по строке табличного поля курсор в сводной таблице позиционируется на колонку экземпляра отчета, соответствующего комбинации ключевых измерений навигатора.

Имеется возможность уменьшить область видимости данных, установив флаги «Отбирать» в нужных строках ключевых измерений. После нажатия кнопки отбора область видимости будет сужена. Это позволяет уменьшить риск ошибок, уменьшить необходимость скроллинга сводной таблицы и ускорить ее отрисовку.
