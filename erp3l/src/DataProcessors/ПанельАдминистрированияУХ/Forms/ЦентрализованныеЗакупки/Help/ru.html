Настройки подсистемы Корпоративные закупки

Графическая схема справки 

С особенностями оформления и использования объектов подсистемы Корпоративные закупки можно ознакомиться на схемах:
- Корпоративные закупки
- Аккредитация поставщиков 
- Подготовка и проведение закупочных процедур 
С другими схемами системы можно ознакомиться по ссылке.

Базовые константы и настройки

  • Использовать корпоративные закупки - опция. Включает / отключает использование подсистемы. 
  • Периодичность закупок - задает периодичность годового плана. Данная настройка используется в документах подсистемы, позволяя вводить данные с указанной периодичностью. 
  • Валюта закупок - валюта, в которой формируется план закупок. Для организаций 223-ФЗ всегда руб.
  • Тип цен для расценки потребности - тип цен номенклатуры, по которому выполняется  расчет стоимости закупок (плановые цены). 
  • Параметры планирования закупок по умолчанию - содержит начальные значения длительности процедуры от способа закупки, используемые объектами подсистемы.
  • Разделять номенклатура по товарным категориям - опция, позволяет выделить часть номенклатуры под ответственность категорийных менеджеров. Требуется настроить матрицу полномочий категорийных менеджеров. Доступ к работе по категориям важен при подготовке плана закупок, отбора документов и анализа отчетов. Обработка "Формирование строк плана закупок" предоставляет пользователям доступ к потребности по товарной категории. Товарная категория добавляется в формы объектов и списки документов, добавляется как отбор или группировка в отчетах. При ручном создании строк плана закупок контроль по матрице полномочий категорийных менеджеров не выполняется, те пользователям доступна вся потребность. Однако, форма строки плана закупок при включенной опции адаптирована для работы с товарными категориями.
  • Матрица полномочий категорийный менеджеров  - назначает по каждой организации для товарной категории ответственного менеджера. Категорийный менеджер имеет доступ к номенклатуре только своей товарной категории, а также к номенклатуре без назначенной товарной категории. Если по товарной категории ответственный не задан, то ни один пользователь не может её увидеть в обработке "Формирование строк плана закупок". Остальной функционал подсистемы таких ограничений не имеет. 
  • Настроить оповещения и напоминания по закупкам -  открывает помощник быстрой настройки оповещений и напоминаний. Для работы с напоминаниями и оповещениями должна быть настроена системная учетная запись и настроен шаблон.

    • Положение о закупках

  • Список правил закупки - открывает список правил закупок, разработанных без привязки к организации. система поддерживает два вида правил, отличающиеся параметрами: для обычных организаций и организаций, ведущих учет по 223-ФЗ.
  • Положение о закупках организации - открывает список организационных единиц с назначенными правилами закупок. Они описывают как закупается так или иная номенклатура для контроля и сокращения трудозатрат при разработке годовой программы закупок. По сути правила закупок, назначенные организации являются электронным аналогом положения о закупках.

    • Аккредитация поставщиков

  • Срок аккредитации по умолчанию - количество месяцев на который выдается аккредитация по умолчанию. Используется документом "Аккредитация поставщика".
  • Длительность рассмотрения заявки на аккредитацию  - количество дней, в течении которых заявка на аккредитацию должна быть проверена. Используется в отчете "Состояние аккредитации". При отправке документа на проверку из личного кабинета поставщика проставляется дата и время. От нее идет отсчет выделенного на проверку времени.

    • Личный кабинет поставщика

  • Использовать личный кабинет - включает доступ внешних пользователей в систему (личный кабинет поставщика). При включении система переходит в режим повышенной безопасности, авторизация для всех пользователей выполняется только по явно указанному логину и паролю. Список выбора пользователей в данном режиме не доступен. 
  • Личный кабинет активирован - при включении флажка "Использовать личный кабинет" система выполняет подключение внешних пользователей (НСИ и Администрирование - Настройки пользователей и прав - флажок "разрешить доступ внешним пользователям"). При этом в системе создается предопределенный внешний пользователь "Саморегистрация поставщиков", под которым выполняется старт авторизации пользователей в личный кабинет. Если подключение прошло без ошибок, то флажок включен. Если флажок выключен можно попытаться принудительно разрешить работу внешних пользователей в панели Администрирования или обратиться к администратору системы.

    • Контроль годового объема закупок у МСП

  • Процент годового объема закупок с участием МСП - константа, используется для проверки контрольных соотношений при утверждении программы закупок, отчетах по закупке. 
  • Процент годового объема закупки только у МСП - константа, используется для проверки контрольных соотношений при утверждении программы закупок, отчетах по закупке.
