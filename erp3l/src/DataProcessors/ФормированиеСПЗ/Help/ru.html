Формирование строк плана закупок

Обработка- мастер предназначена для создания строк плана закупок из плана потребности по по правилам, прописанным в положении о закупках организации, а также актуализации созданных строк плана при изменении плана потребности.     

Графическая схема справки

С особенностями оформления и использования объектов подсистемы Корпоративные закупки можно ознакомиться на схемах:

-Корпоративные закупки
-Подготовка и проведение торгов
-Инвестиционные проекты

С другими схемами системы можно ознакомиться по ссылке.

Шаг 1 Уточнение потребностей

Обработка на вход принимает потребность, зарегистрированную операционными планами по отборам ("Период", "Учет по 223-ФЗ"), установленным на первой странице, а также доступной пользователям по матрице полномочий категорийных менеджеров (если включена константа "разделять номенклатуру по товарным категориям"). До начала работы с обработкой необходимо настроить номенклатуру по которой имеются аналоги (размещается в "Корпоративные закупки" - "Настройка замены номенклатуры"). Это регистр сведений содержащий возможные аналоги. Если он пустой, то система считает, что аналогов для замены нет. 

Пользователю доступны следующие действия до перехода к созданию строк плана закупок:

  • Позиции потребности не классифицированные по ОКПД2 - действие отображается в списке только при включенном параметре отбора "Учет по 223-ФЗ". Анализирует потребность (справочник "Номенклатура") зарегистрированную операционными планом и показывает количество номенклатуры с незаполненными кодами ОКПД2. нажав на количество пользователь переходит к списку номенклатуры для заполнения данных. Код ОКПД2 является обязательным для закупок по 223-ФЗ и без него невозможно создать корректные строки плана закупок. 
  • Необходимо заменить на унификаты - замена на унификаты выполняется автоматически в документе Операционный план. Данный пункт обрабатывает ситуацию, когда информацию об унификате ввели после ввода операционного плана. Такие позиции можно заменить до начала работы по созданию строк плана. 
  • Возможна замена на аналоги - показывает количество позиций, по которым имеются аналоги. Замена выполняется в два шага. Сначала подбор и закрепление аналога для замены по гиперссылке "Настроить", а вторым шагом замена "Обработать".
  • Возможно обновление ранее созданных строк план закупок - позволяет актуализировать ранее созданные строки плана по изменившейся потребности, зарегистированной операционным планом. Если строку плана закупок нельзя актуализировать, то в строке плана нужно установить флажок "Запретить автоматическое изменение". Строки плана, по которым стартовала закупочная процедура также не подлежат актуализации.
  • Организации, по которым отсутствуют положения о закупке - строки плана формируются по правилам, прописанным в положении о закупке. Наличие положения о закупках обязательно для работы данной обработки. Если по организации отсутствует положение о закупке, то положение нужно ввести. В противном случае потребности данной организации не будут доступны на следующих шагах мастера.

В нижней части формы располагаются кнопки перехода вперед/назад, список черновиков и прочие настройки. Обработка в качестве финального результата выдает точные копии строк плана закупок, для предварительной проверки и правки пользователями. В терминах системы "Черновики СПЗ". Такой подход позволяет не замусоривать систему удаленными объектами. Черновики можно удалить и сформировать заново. И только когда все проверено, можно нажать кнопку "Сформировать СПЗ из черновика", по которой будут сформированы настоящие объекты системы - строки плана закупок.

Шаг 2 Подбор потребностей

Форма поделена на три области: "информация о потребности", "отборы" и "корзина". В обработку потребность может поступать частями, а может целиком. Потребность добавляется в корзину при двойном клике мышкой по строке потребности. Для добавления всей потребности в корзину используется кнопка "Выбрать все". Потребность в таблице свернута по периодам, а данный агрегированы. Для анализа потребности по периодам нажмите на кнопку "По периодам". Обработка поддерживает два режима работы, ручной и пакетный (автоматический). 

В ручном режиме пользователь набирает в корзину потребность, которая будет закупаться вместе одним лотом. При этом при добавлении в корзину выполняется проверка совместимости номенклатуры с правилами положения о закупке. Если правила противоречивы, отрывается форма отчета "Потребности с противоречиями", в которой можно провести анализ для исключения из обработки несоответствующей номенклатуры. При проверке проверяется, чтобы номенклатура имела единый способ закупки, форму проведения и другие параметры, жестко определяющие правило закупок. Организации, составляющие план по 223-ФЗ, могут формировать план только в рамках одного юридического лица и его обособленных подразделений. Для остальных организаций такого ограничения нет. 

В пакетном режиме пользователь набирает в корзину потребность, по которой нужно быстро подготовить план закупок в соответствие с правилами, прописанными в положении о закупках. Автоматически обрабатывается вся потребность, добавленная в корзину и в результате создаются черновики строк плана закупок. Пользователь на 4 шаге может уточнить группировку номенклатуры в строки плана, но вся потребность будет проверена на непротиворечивость по правилам положения о закупках.

Шаг 3 Формирование строк плана закупок (ручной режим)

Данный шаг мастера отображается только при ручном режиме работы. Пользователь принимает решение: добавить потребность в созданный ранее черновик строки плана, добавить к существующей строке плана, отнести на действующий договор (будет создана новая версия договора) или создать строку плана закупок. Для удобства подбора реализованы гибкие отборы, позволяющие бствро посмотреть объекты системы до совпадения по номенклатуре (точное соответствие), товарной категории или ОКПД2 (подбор совпадающих объектов в указанных разрезах). Для выбора объекта дважды щелкните по нему (выставиться "галочка"). 

Шаг 4 Формирование строк плана закупок (автоматический режим) 

Данный шаг мастера отображается только в пакетном режиме. Пользователь принимает решение об объединении потребности в строки плана. В верхней части формы отображается отчет с потребностью корзины для обработки. В нижней части формы расположена управляющая панель "Разрезы формирования строк плана закупок". 

  • Период [Год | Полугодие | Квартал | Месяц] - отвечает за деление плана по периодам. Так, если выбрать квартал, то годовой план будет разделен на четыре части. Будет спланировано четыре закупки одной и той же номенклатуры потребности по одной в каждом квартал. 
  • Номенклатура [Номенклатура | Товарная категория | ОКПД2] - при создании строк плана объединяет номенклатуру до указанного разреза группировки. Например, если выбрать "ОКПД2", то номенклатура с одинаковым ОКПД2 будет собрана в одной строке плана закупок. При включенной константе "Разделять номенклатуру по товарным категориям" обязательным разрезом группировки будет товарная категория. Причем она будет обязательным разрезом для номенклатуры или ОКПД2. Например, если выбран разрез ОКПД2, то при выключенной константе будут созданы строки плана по ОКПД2, а при включенной константе они будет поделены еще и по товарным категориям.
  • Группа тонких параметров ("Способ закупки","Организатор",  "Место поставки","Филиал", "Организация заказчик", "Проект", "Менеджер") позволяет задать дополнительные разрезы деления номенклатуры по строка плана. Однако, данный набор параметров работает только на "мягкие правила" то есть правила, разрешающие изменение значение параметра. Например, в положении о закупках для номенклатуры способ закупки задан как "конкурс", но задан без замочка (мягкое правило). Это означает, что пользователи могут поменять способ закупки на любой другой.  А значит обработка может объединить в одну строку, номенклатуру с разными способами закупок, заданными "мягко", но никогда не нарушит жестко заданные правила (правила с закрытым замочком).  

Шаг 5 Предварительный просмотр

На данном шаге пользователь имеет возможность посмотреть результаты обработки в виде созданных черновиков плана закупки, проверить, заполнить недостающие реквизиты и даже поменять способ закупки и другие параметры, выделенные в черновике зеленым фоном. Эти параметры можно поменять, так как в положении о закупках задано "мягко". Жестко заданные параметры пользователь поменять не может. Они выделены желтым фоном.  

Шаг 6 Создание строк плана закупок

Доступен из списка черновиков. На данном шаге из черновиков можно создать строки плана.
